# 文档模板参考

## 系统GDD模板

按行业标准的GDD结构，每个系统内部包含7个部分：

```
# [系统名称] 设计文档

## 目录
1. Objectives（目标）
2. Overview（概述）
3. Mechanics（机制）
4. Data Flow（数据流）
5. UI（界面）
6. Edge Cases（边界条件）
7. Configuration（配置表）

---

## 1. Objectives（目标）
- 系统目标：[这个系统要解决什么问题]
- 体验目标：[玩家应该感受到什么]
- 成功标准：[如何判断系统设计成功]

## 2. Overview（概述）
- 系统定位：[在整个游戏中的位置]
- 核心机制：[用一段话描述系统如何运作]
- 玩家视角：[玩家如何理解和使用这个系统]

## 3. Mechanics（机制）
### 3.1 核心规则
- 规则1：[具体规则描述]
- 规则2：[具体规则描述]
- ...

### 3.2 输入/输出
| 输入 | 处理逻辑 | 输出 |
|------|---------|------|
| [玩家操作/系统事件] | [处理规则] | [结果] |
| ... | ... | ... |

### 3.3 公式定义
- [公式1]：[公式描述]
  - 变量说明：[每个变量的含义]
  - 取值范围：[每个变量的合理范围]

## 4. Data Flow（数据流）
### 4.1 数据来源
- [数据项1]：来自[系统A]
- [数据项2]：来自[系统B]

### 4.2 数据输出
- [数据项1]：输出到[系统C]
- [数据项2]：输出到[系统D]

### 4.3 数据流图
[用文字描述数据流关系，或用Mermaid语法绘制流程图]

## 5. UI（界面）
### 5.1 界面布局
- 主界面：[描述界面元素和布局]
- 子界面：[描述子界面和跳转逻辑]

### 5.2 交互流程
1. [步骤1]：[玩家操作] → [系统响应]
2. [步骤2]：[玩家操作] → [系统响应]
3. ...

## 6. Edge Cases（边界条件）
| 边界条件 | 处理方式 |
|---------|---------|
| [条件1] | [处理方式] |
| [条件2] | [处理方式] |
| ... | ... |

## 7. Configuration（配置表）
### 7.1 配置表结构
| 字段名 | 类型 | 说明 | 示例值 |
|-------|------|------|-------|
| [字段1] | [int/float/string] | [说明] | [示例] |
| [字段2] | [int/float/string] | [说明] | [示例] |
| ... | ... | ... | ... |

### 7.2 可调参数
| 参数名 | 当前值 | 取值范围 | 说明 |
|-------|-------|---------|------|
| [参数1] | [值] | [范围] | [说明] |
| [参数2] | [值] | [范围] | [说明] |
| ... | ... | ... | ... |
```

## 配置表模板

### 经济系统配置表
```
| 字段名 | 类型 | 说明 | 示例值 |
|-------|------|------|-------|
| currency_type | string | 货币类型 | "gold"/"diamond" |
| source_id | int | 产出来源ID | 1001 |
| source_amount | int | 产出数量 | 100 |
| sink_id | int | 消耗去向ID | 2001 |
| sink_amount | int | 消耗数量 | 50 |
| exchange_rate | float | 兑换比率 | 1.2 |
```

### 成长系统配置表
```
| 字段名 | 类型 | 说明 | 示例值 |
|-------|------|------|-------|
| level | int | 等级 | 1 |
| exp_required | int | 所需经验 | 100 |
| exp_cumulative | int | 累计经验 | 100 |
| stat_hp | int | 生命值 | 1000 |
| stat_atk | int | 攻击力 | 100 |
| stat_def | int | 防御力 | 50 |
```

### 概率系统配置表
```
| 字段名 | 类型 | 说明 | 示例值 |
|-------|------|------|-------|
| item_id | int | 物品ID | 3001 |
| rarity | string | 稀有度 | "common"/"rare"/"epic"/"legendary" |
| drop_rate | float | 掉落概率 | 0.5 |
| guarantee_count | int | 保底次数 | 10 |
```

## 验证报告模板

```
## 逻辑验证报告

### 1. 逻辑自洽性检查
| 检查项 | 结果 | 说明 |
|-------|------|------|
| 核心循环是否完整 | ✅/❌ | [说明] |
| 系统间依赖是否单向 | ✅/❌ | [说明] |
| 公式是否自洽 | ✅/❌ | [说明] |
| 边界条件是否覆盖 | ✅/❌ | [说明] |

### 2. 耦合度检查
| 系统 | 依赖系统数量 | 被依赖系统数量 | 耦合度评估 |
|------|-------------|---------------|-----------|
| [系统A] | [数量] | [数量] | 低/中/高 |
| [系统B] | [数量] | [数量] | 低/中/高 |
| ... | ... | ... | ... |

**耦合度评估标准**：
- 低：依赖≤2个系统
- 中：依赖3-5个系统
- 高：依赖>5个系统

### 3. 边界条件覆盖度
| 边界条件类型 | 是否覆盖 | 说明 |
|-------------|---------|------|
| 数值边界（最大值/最小值） | ✅/❌ | [说明] |
| 状态边界（异常状态） | ✅/❌ | [说明] |
| 时间边界（超时/并发） | ✅/❌ | [说明] |
| 用户边界（非法输入） | ✅/❌ | [说明] |

### 4. 改进建议
- [建议1]：[具体说明]
- [建议2]：[具体说明]
- ...
```
