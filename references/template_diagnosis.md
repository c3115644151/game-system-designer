## 使用说明

1. **步骤0初始化**：创建文档，命名为`[系统名称]_问题诊断.md`
2. **步骤1问题拆解**：填充第1章节
3. **步骤2根因定位**：填充第2章节
4. **步骤3修改方案**：填充第3章节

## AI辅助要点

- AI基于问题现象追溯结构性原因
- AI需要明确标注推断的不确定性
- 根因分析需要证据支撑，不能凭空猜测
- 解决方案需要人类评估可行性（技术/资源/时间约束）
- 风险评估需要人类基于经验判断
- 验证计划需要人类确定具体的验证方法和成功标准

## 诊断框架

### 常见问题类型
1. **激励不一致**：玩家行为与设计目标不一致
2. **过度耦合**：系统间依赖过多，修改一个影响多个
3. **验证门缺失**：关键节点缺少验证，导致问题累积
4. **数值失衡**：产出/消耗不平衡，成长曲线不合理
5. **体验断裂**：核心循环各层级之间缺乏因果关系

### 根因分析方法
1. **5 Whys**：连续追问"为什么"，追溯到根本原因
2. **鱼骨图**：从人、机、料、法、环多个维度分析原因
3. **故障树**：从顶事件向下分解，找出所有可能的原因
4. **数据驱动**：通过数据分析定位问题的根本原因
