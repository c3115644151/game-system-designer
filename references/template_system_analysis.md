# 现有系统分析报告模板

基于逆向GDD（Reverse GDD）行业标准，适配AI辅助分析场景。

## 文档结构

```
# [游戏名称] - [系统名称] 分析报告

## 目录
1. 系统概述（System Overview）
2. 机制拆解（Mechanics Breakdown）
3. 系统交互（System Interactions）
4. 设计评价（Design Evaluation）

---

## 1. 系统概述（System Overview）

### 1.1 系统定位
- 游戏类型：[游戏所属品类]
- 系统类型：[核心系统/支撑系统/辅助系统]
- 目标受众：[这个系统面向的玩家群体]
- 核心目的：[这个系统在游戏中承担什么功能]

### 1.2 核心体验
- 设计意图：[设计师希望玩家获得什么体验]
- 实际体验：[玩家实际感受到的体验]
- 体验差距：[设计意图与实际体验的差距分析]

### 1.3 系统边界
- 包含功能：[系统负责的功能列表]
- 不包含功能：[明确排除的功能]
- 关联系统：[与本系统有交互的其他系统]

---

## 2. 机制拆解（Mechanics Breakdown）

### 2.1 核心循环（四层嵌套）

#### 秒级操作（Core Action）
- 核心动作：[玩家最频繁的操作]
- 操作频率：[每秒/每几秒一次]
- 操作反馈：[操作后的即时反馈]
- 体感评价：[操作手感、流畅度、满足感]

#### 分钟级活动（Activity）
- 活动定义：[由多个秒级操作组成的完整活动]
- 活动时长：[预计持续时间]
- 活动结构：[开始→过程→结束的完整流程]
- 节奏评价：[活动节奏是否舒适]

#### 会话级目标（Session Goal）
- 目标定义：[一次游戏会话中玩家追求的目标]
- 目标类型：[收集/成长/探索/社交/成就]
- 目标达成感：[目标达成时的满足感评价]

#### 元循环（Meta Loop）
- 循环定义：[跨会话的长期目标循环]
- 进度类型：[角色成长/资源积累/内容解锁]
- 长期动力：[玩家持续参与的动力来源]

### 2.2 子系统拆解（System Stack）

|-----------|---------|------|------|---------|
| [子系统1] | [一句话描述] | [接收什么] | [输出什么] | [依赖谁] |
| [子系统2] | [一句话描述] | [接收什么] | [输出什么] | [依赖谁] |

### 2.3 核心规则

- 规则2：[具体规则描述]
- ...

### 2.4 数值框架

## 使用说明

1. **步骤0初始化**：创建文档，命名为`[游戏名称]_[系统名称]_分析报告.md`
2. **步骤1系统拆解**：填充第1、2章节
3. **步骤2数值分析**：补充2.4数值框架部分
4. **步骤3设计意图**：填充第3章节和4.3设计意图推断
5. **步骤4评估建议**：填充4.1-4.5评价部分

## AI辅助要点

- AI基于公开信息（游戏文档、玩家社区、视频等）进行初步拆解
- 体感相关的部分（如操作手感、节奏舒适度）需要人类实际体验后补充
- 设计意图推断需要明确标注不确定性程度
- 改进建议需要人类评估可行性（技术/资源/时间约束）
